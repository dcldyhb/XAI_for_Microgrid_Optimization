## 二、项目背景与研究意义

在台风、暴雨、高温等极端天气场景下，微电网中的光伏、风机和储能等分布式能源会出现不同程度的出力降额，系统可行运行域随之收缩，电压越限和调度不可行风险明显上升。传统优化方法在这类强时变、强不确定工况下通常依赖较准确的模型和较高的在线计算开销，而纯数据驱动强化学习虽然具备快速决策能力，但存在可解释性不足、训练效率波动较大、难以说明“为什么这样调度”的问题。

本项目围绕“安全性、经济性、可解释性”三个目标，尝试将符号回归与深度强化学习结合，构建一种适用于 IEEE 33 节点微电网的极端天气运行优化方法。项目的核心思路是：先通过物理仿真构造动作灵敏度标签，再利用 PySR 学习显式公式，将其作为结构先验嵌入 Soft Actor-Critic, SAC 控制器，从而在保留快速在线决策能力的同时，提高控制策略的可解释性。

本项目的研究意义主要体现在以下几个方面：

1. 面向极端天气条件下的微电网运行问题，探索兼顾安全与经济的控制方法。
2. 将符号回归引入强化学习控制流程，增强策略生成过程的可解释性。
3. 通过“物理标签 + 可解释先验 + 策略学习”的组合路线，为后续开展面向复杂配电网场景的可解释智能调度研究提供基础。

## 三、项目目标

本项目立项时的主要目标包括：

1. 搭建极端天气下 IEEE 33 节点微电网仿真环境，构建可用于学习与分析的数据集。
2. 设计面向电压安全的灵敏度标签生成方法，建立状态到灵敏度的监督学习数据。
3. 使用 PySR 对灵敏度关系进行建模，提取可解释的显式公式。
4. 将符号先验嵌入 SAC 控制器，实现安全经济运行决策。
5. 对比无先验 SAC 与引入符号先验的 SAC，评估方法在安全性、经济性与可解释性方面的表现。

## 四、研究内容与技术路线

### 4.1 总体技术路线

项目采用“数据构造 - 物理标注 - 符号建模 - 强化学习控制 - 对比验证”的技术路线。具体流程如下：

1. 基于 IEEE 33 节点微电网模型和 Pandapower 仿真环境，构建极端天气运行样本。
2. 对天气、负荷、储能状态、电压、线路负载等特征进行整理，并生成扩充数据集。
3. 在各样本工作点附近对储能动作进行微小扰动，计算尾部电压对动作的局部灵敏度，作为监督学习标签。
4. 采用 PySR 对“状态变量 x 到动作灵敏度 y”的关系进行分量级建模，得到一组闭式表达式。
5. 将符号回归得到的结构调制向量输入 SAC 网络，在 actor 和 critic 的隐藏层中进行 FiLM 风格调制。
6. 对比带符号先验与不带先验的 SAC，在安全性与经济性方面进行分析。

### 4.2 项目系统结构

当前代码已形成较完整的模块化结构，主要包括：

1. `augment_ieee33.py`：用于样本扩充、局部灵敏度计算与安全裕度标签生成。
2. `PySR_gamma/`：用于数据划分、PySR 训练与显式公式导出。
3. `SAC_gama/`：包含带 `gamma` 调制的 actor、critic 与经验回放机制。
4. `main_gamma.py`：使用 PySR 生成公式进行训练。
5. `main_sac.py`：作为无符号先验的对照实验入口。

## 五、已完成的主要工作

### 5.1 完成了极端天气微电网数据构建

项目已基于原始 IEEE 33 节点数据与环境扰动机制，生成增强后的物理数据集，并完成训练集与测试集划分。当前本地数据规模如下：

1. 增强后全量数据集：19200 条样本。
2. 训练集：15360 条样本。
3. 测试集：3840 条样本。

数据特征包括温度、风速、光照、降雨、最小电压、最大电压、线路负载率、网损、5 个储能荷电状态、负荷需求、小时信息和星期信息等。

### 5.2 完成了动作灵敏度标签构造

项目在每个样本工作点附近，对 5 个电池有功动作分别进行微小正负扰动，并计算尾部电压指标的变化量，近似得到动作灵敏度标签 `sens_action_batt_0` 到 `sens_action_batt_4`。这一部分工作将原本不可直接观察的“控制效果”转化为了可监督学习的目标，为后续符号回归建模提供了基础。

从方法上看，这一步相当于在当前实现中以有限差分方式近似论文中的 Jacobian 灵敏度分析，属于论文前半部分思想的工程化简化版复现。

### 5.3 完成了基于 PySR 的显式公式学习

项目已经完成对 5 个储能动作灵敏度目标的 PySR 建模，并自动导出 `gamma_calculator.py` 供强化学习阶段直接调用。当前模型设置采用：

1. 二元运算符：`+`、`-`、`*`、`/`
2. 一元运算符：`square`、`cube`
3. 最大表达式规模：25
4. 复杂度惩罚参数：0.001

该设置体现了“复杂度 - 精度”的平衡思想，即在保证公式尽量简洁、可读、可部署的前提下，尽量保留对灵敏度变化趋势的拟合能力。

### 5.4 完成了符号先验引导的 SAC 控制器实现

项目已实现将 PySR 输出的结构调制向量 `gamma` 输入到 SAC 的 actor 和 critic 网络中，在隐藏层进行动态调制。与传统仅依赖状态输入的 SAC 相比，该实现使策略网络在决策时能够参考“当前状态下哪些动作更可能提升安全裕度”的先验信息，从而增强决策的结构性和可解释性。

### 5.5 完成了基线对照实验

目前已完成多组 `PySR-guided SAC` 与 `SAC-only` 的训练摘要保存，能够对两类方法在相同训练步数下的安全性与经济性进行对照分析。

## 六、阶段性成果与结果分析

### 6.1 PySR 公式学习结果

当前得到的 5 个目标公式测试集 `R2` 结果如下：

| 目标信号 | 测试集 R2 |
| :--- | ---: |
| `sens_action_batt_0` | 0.2368 |
| `sens_action_batt_1` | 0.0439 |
| `sens_action_batt_2` | 0.2584 |
| `sens_action_batt_3` | 0.1139 |
| `sens_action_batt_4` | -0.0010 |

从当前结果看，`sens_action_batt_0` 和 `sens_action_batt_2` 的公式具备一定趋势拟合能力，`sens_action_batt_1` 和 `sens_action_batt_3` 的解释能力较弱，`sens_action_batt_4` 尚未形成有效可用的闭式规律。这说明当前标签构造与特征设计对部分动作维度有效，但整体仍有较大提升空间。

从相关性分析结果看，当前灵敏度标签与以下变量的相关性较强：

1. `grid_load_demand`
2. `elec_vmin`
3. `hour_of_day`

这表明当前版本中，动作灵敏度主要反映了“负荷压力 - 最小电压水平 - 运行时段”对尾部电压改善空间的影响。天气变量也进入了部分公式，但其主导程度暂未完全体现出来。

### 6.2 复杂度、Pareto 曲线与可解释性分析

PySR 的核心优势在于能够同时给出多组不同复杂度的候选公式。理论上，横轴为复杂度、纵轴为损失的 Pareto 曲线可以帮助我们选择位于“拐点区域”的公式，即在表达式不过分膨胀的前提下保留主要趋势信息。

对于本项目而言，这种平衡具有直接工程意义：

1. 公式越简单，越容易解释 `x` 与 `y` 的关系，也越方便后续嵌入控制器。
2. 公式越复杂，理论上可能拟合更充分，但也更容易过拟合，且难以说明物理含义。
3. 因此，本项目并不追求“最复杂、最贴合训练数据”的表达式，而是优先选择“能够揭示主要关系、且复杂度可控”的可部署公式。

在当前代码实现中，这一思路体现在 `maxsize=25` 与 `parsimony=0.001` 的参数设置上。虽然本地结果尚未系统导出完整 Pareto 曲线图，但方法论已经具备，后续只需补充可视化脚本即可形成标准图表。

### 6.3 `x` 与 `y` 关系的解释

本项目中：

1. `x` 表示可观测运行状态，包括天气、负荷、电压、网损、SOC 和时序变量。
2. `y` 表示某个储能动作对尾部电压安全指标的边际影响，即灵敏度。

因此，PySR 学到的并不是“状态直接映射到最终动作”，而是“状态如何影响各动作对安全指标的作用强弱”。这个中间层关系本身就是可解释的：

1. 当负荷上升、最小电压下降时，系统通常处于更脆弱工况，动作灵敏度会发生明显变化。
2. 不同时间段和不同天气条件会改变可再生能源出力与系统潮流分布，进而影响储能动作对弱电压区域的支撑效果。
3. SAC 不再只依赖黑箱网络从头学习动作，而是参考这些显式灵敏度规律进行策略生成。

换言之，本项目通过 PySR 先解释“在什么状态下什么动作更有效”，再让 SAC 据此学习“应该如何组合动作”，从而提高了策略的可解释性。

### 6.4 强化学习对照结果

根据当前仓库中可直接读取的训练摘要文件，在若干组 `PySR-guided SAC` 与 `SAC-only` 对照实验中，两类方法都实现了零电压越限率，但引入 PySR 先验后，经济性指标表现出一定改善。

以两组可直接对照的训练结果为例：

| 对照组 | `gamma_mode` | 训练步数 | 平均电压越限 | 越限步占比 | 能量成本（RMB） |
| :--- | :--- | ---: | ---: | ---: | ---: |
| `2026-03-30_10-18-15_IEEE33Env_Gaussian_SAC_ONLY` | `none` | 5016 | 0.0 | 0.0 | 1438054.50 |
| `2026-03-30_10-18-15_IEEE33Env_Gaussian_PYSR` | `pysr` | 5016 | 0.0 | 0.0 | 1422258.25 |
| `exp_sac_tuned_v6` | `none` | 5016 | 0.0 | 0.0 | 1453114.00 |
| `exp_gamma_tuned_v6` | `pysr` | 5016 | 0.0 | 0.0 | 1441548.12 |

由此可见，在当前本地可复现实验中，引入 PySR 先验后，能量成本下降约 0.8% 至 1.1%，同时保持零电压越限。说明符号先验在当前实现中已开始发挥作用，但改善幅度仍明显低于论文中理想化或更完整设定下的结果，仍需要进一步优化。

## 七、项目创新点

本项目的主要创新点包括：

1. 将符号回归 PySR 引入微电网强化学习控制流程，而不是仅将其用作独立建模工具。
2. 采用“状态到动作灵敏度”的中间建模方式，把可解释先验注入策略学习过程。
3. 使用结构调制向量对 actor 和 critic 隐层进行调制，使先验知识以轻量方式参与决策。
4. 在工程实现上打通了“数据增强 - 物理标签 - 显式公式 - RL 控制”完整链路。

## 八、存在的问题与不足

虽然项目已经完成核心框架搭建和初步对照实验，但仍存在以下不足：

1. 当前灵敏度标签是通过有限差分近似得到的，仅覆盖 5 个电池有功动作，尚未完整覆盖论文中的全动作空间。
2. 当前灵敏度标签波动范围较小，导致部分目标的 `R2` 较低，说明监督信号仍然偏弱。
3. 目前尚未系统输出完整的 PySR Pareto 曲线图、损失 - 复杂度拐点图和更规范的精度分析图。
4. 当前实验中安全指标几乎均为零越限，说明场景挑战性有待进一步提高，也可能导致不同方法间差异被削弱。
5. 当前成果更适合作为“前半部分方法验证 + 系统原型实现”，距离完整论文级复现仍有一定差距。

## 九、下一步工作计划

针对上述问题，后续可从以下几个方向继续完善：

1. 将当前有限差分标签升级为更接近论文设定的 Jacobian 灵敏度标签。
2. 将动作空间从当前 5 维储能有功动作扩展到储能无功、光伏无功和风机无功等更多控制量。
3. 补充 PySR 候选公式库与 Pareto 曲线自动可视化脚本，形成完整的复杂度 - 精度分析链路。
4. 引入更严苛的极端天气工况与更高强度越限场景，提高方法区分度。
5. 在保持可解释性的基础上进一步优化 SAC 调制强度、训练策略和奖励设计，提升经济性改进幅度。

## 十、结论

总体而言，本项目已经完成了立项阶段提出的核心技术框架与主要工程任务，成功实现了极端天气下微电网运行优化问题中“PySR 可解释先验 + SAC 控制策略”的基本闭环。

现阶段成果表明：

1. 项目已具备完整的数据构建、标签生成、符号建模和强化学习控制能力。
2. PySR 公式能够在一定程度上刻画运行状态到动作灵敏度之间的关系，为策略网络提供可解释结构先验。
3. 与无先验 SAC 相比，带符号先验的控制器在本地实验中表现出一定经济性改善，并保持零电压越限。

因此，本项目已经形成较完整的研究原型和实验基础，达到了“完成核心技术路线验证、形成阶段性研究成果、具备后续扩展潜力”的结项要求。
